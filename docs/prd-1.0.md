## 1. 제품 한 줄 정의

시골·읍면 지역에서 다시 일하고 싶은 50~70대 중장년/시니어 구직자가 복잡한 온라인 등록 없이 자신의 경력과 조건을 쉽게 남기고, 가까운 지역의 믿을 수 있는 일자리와 연결되도록 돕는 채용 플랫폼입니다.

## 2. 문제 정의

시골·읍면 지역의 중장년/시니어 구직자는 일할 의지가 있어도 일자리 정보가 주민센터, 지인 소개, 현수막, 밴드, 워크넷 등에 흩어져 있고 온라인 회원가입이나 이력서 작성이 어려워 실제 지원까지 이어지지 못합니다.

그 결과 가까운 지역의 일자리 기회를 놓치고, 지인 소개나 우연히 접한 정보에만 의존하게 되어 시간과 소득 기회를 잃습니다.

## 3. 핵심 고객

- 1차 고객: 시골·읍면 지역에서 다시 일하고 싶은 50~70대 중장년/시니어 구직자
- 집중 대상: 구직자/인재 측
- 주요 상황: 일할 의지는 있지만 온라인 등록과 정보 탐색이 어렵고, 지역 일자리 정보가 흩어져 있음
- 원하는 결과: 본인이 할 수 있는 일과 원하는 근무 조건을 쉽게 등록하고, 가까운 지역의 믿을 수 있는 일자리에 지원하거나 연결됨

## 4. 가치 가설

시골·읍면 지역 중장년/시니어 구직자의 일자리 연결 단절 고통을 쉬운 등록과 지역 기반 매칭으로 줄입니다.

### 비용을 지불할 가능성이 있는 대상

- 지자체
- 공공기관
- 지역 일자리 운영 기관
- 지역 기반 인력이 필요한 사업자

### 수익 모델 가설

- 공고 게재료
- 월 구독료
- 지역 일자리 관리 비용

가격 가설: 지자체와 공공기관이 시니어 인력을 지역 내 단기 일자리에 쉽게 연결해주면 공고 게재 및 관리 비용으로 월 구독료를 낼 것입니다.

## 5. MVP 범위

### 핵심 기능

1. 음성인식 기반 경력 입력
   - 사용자가 긴 온라인 이력서를 직접 작성하지 않고 말로 경력과 희망 조건을 남길 수 있게 합니다.

2. 사진 및 주민등록 인식 기반 간편 가입
   - 복잡한 입력을 줄이고 초기 등록 장벽을 낮춥니다.

3. 지역 기반 일자리 매칭
   - 사용자의 지역과 조건을 기준으로 가까운 일자리 정보를 보여주고 연결합니다.

### MVP의 핵심 기능 1개

주민센터, 복지관 제휴 지역 기반 일자리 매칭

### 이번 버전에서 제외할 기능

- 자녀가 부모님 이력을 대신 등록하고 알림을 받는 기능

## 6. 첫 사용 흐름

1. 사용자는 복지관 또는 주민센터 안내를 통해 서비스에 접속합니다.
2. 간편 회원가입을 시작합니다.
3. 사진 및 주민등록 인식 기능으로 기본 정보를 빠르게 입력합니다.
4. 사용자는 말로 본인의 경력과 가능한 업무를 설명합니다.
5. 서비스는 해당 정보를 바탕으로 지역 기반 일자리 후보를 보여줍니다.
6. 사용자는 가까운 지역의 일자리 정보를 확인하고 지원 또는 연결을 진행합니다.

## 7. 주요 화면 초안

### 화면 1. 간편 등록 시작

- 가치: 온라인 이력서 작성이 어렵다는 문제를 줄임
- 행동: 사용자가 구두로 이력 정보를 남김
- 기능: 음성인식
- 구성요소: 간편 회원가입, 구술 이력 작성, 지역 일자리 매칭 진입
- 다음 행동: 사진 및 주민등록 인식으로 기본 정보 입력

### 화면 2. 기본 정보 인식

- 가치: 간편한 회원가입
- 행동: 사진 및 주민등록 스캔
- 기능: 사진 및 주민등록 인식 기능
- 목적: 사용자가 직접 입력해야 하는 항목을 줄임

### 화면 3. 지역 일자리 매칭

- 가치: 일자리 정보를 한눈에 확인
- 행동: 지역 단위 일자리 지도 또는 목록 확인
- 기능: 지역 기반 일자리 매칭
- 목적: 가까운 지역의 일자리 후보를 확인하고 지원 또는 연결

## 9. 데이터 기준

데이터 이름은 구현 전에 `docs/data-guide.md`의 공통 이름 사전을 먼저 확인합니다.

새 컬렉션, 필드, 상태값, 역할 값이 필요하면 바로 코드에 추가하지 않고 먼저 공통 이름 사전에 등록합니다. 같은 의미의 데이터를 `age`, `나이`, `연령`처럼 서로 다르게 만들지 않는 것이 원칙입니다.

현재 `docs/data-guide.md`는 기본 채용 플랫폼 역할과 데이터 구조를 출발점으로 갖고 있으므로, 시니어 지역 일자리 MVP 구현 전에 아래 데이터 이름을 먼저 정리해야 합니다.

- 시니어 구직자 프로필
- 경력 구술 내용
- 거주 또는 희망 근무 지역
- 지역 일자리
- 주민센터/복지관 제휴 기관
- 매칭 상태
- 지원 또는 연결 상태

## 10. 기술 기준

- 프론트엔드: React, TypeScript, Vite
- 실행 도구: Bun 권장, npm 사용 가능
- 배포/연동: Firebase 기준
- Firebase SDK 값: `.env`에만 저장하고 문서에는 기록하지 않음
- 공유용 환경변수 문서: `.env.example`

Firebase 기능을 실제 코드에서 사용할 때는 `docs/firebase-guide.md` 기준에 따라 연결 코드를 한 곳에서 관리합니다.
